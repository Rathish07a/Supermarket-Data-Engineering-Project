# Sales Performance Analysis

## Overview

The Sales Performance Analysis module evaluates supermarket sales data to provide business insights into store performance, product sales, and revenue trends.

The analysis is performed using the Gold Layer datasets, which contain clean, validated, and business-ready data generated through the Medallion Architecture.

The results support management in monitoring sales performance, identifying top-performing stores and products, and making data-driven business decisions.

---

# Objectives

The Sales Performance Analysis aims to:

- Measure total revenue generated by each store.
- Identify top-selling products.
- Analyze monthly sales trends.
- Compare store performance.
- Calculate key business metrics.
- Generate business-ready analytical reports.

---

# Gold Layer Datasets

The analysis uses the following Gold Layer tables:

- Gold Sales Performance
- Gold Monthly Sales Performance
- Gold Product Performance

These datasets are generated from the Silver Layer after applying aggregations and business calculations.

# Data Processing Workflow

The Sales Performance Analysis follows these steps:

1. Read Gold Layer Delta tables.
2. Calculate revenue metrics.
3. Aggregate product sales.
4. Generate store performance metrics.
5. Analyze monthly sales trends.
6. Export reports as CSV files.

# Sample SQL Queries

### Total Revenue by Store

```sql
SELECT
    store_name,
    SUM(total_amount) AS total_revenue
FROM gold_sales_performance
GROUP BY store_name
ORDER BY total_revenue DESC;
```

---

### Top Selling Products

```sql
SELECT
    product_name,
    SUM(quantity_sold) AS total_quantity
FROM gold_product_performance
GROUP BY product_name
ORDER BY total_quantity DESC;
```

---

### Monthly Sales Trend

```sql
SELECT
    sales_month,
    SUM(total_revenue) AS monthly_revenue
FROM gold_monthly_sales_performance
GROUP BY sales_month
ORDER BY sales_month;
```

# Sample PySpark Implementation

The Gold Layer data is loaded into PySpark for further analysis and report generation.

```python
sales_df = spark.read.format("delta").load(gold_sales_path)
```

Revenue calculations are performed using PySpark aggregations.

```python
from pyspark.sql.functions import sum

sales_summary = sales_df.groupBy("store_name") \
    .agg(sum("total_amount").alias("total_revenue"))
```
